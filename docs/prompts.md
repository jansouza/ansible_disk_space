# Plan Prompt

```
Create a implementation plan for an Ansible disk space management project with:
- Two roles: disk_verification and filesystem_cleanup
- Two playbooks: verify_disk_space.yml and remediate_disk_space.yml
- Cleanup targets: /tmp, /var/log, package cache, and systemd journals
- Support for Ubuntu, CentOS, and RHEL
- Inventory example and README

The plan must include:
- Ansible best practices
- Safety considerations and dry-run behavior
- Testing and validation approach
- Suggested implementation order
```

# Execution Prompt

```
Implement the approved Ansible plan for disk space management.

Requirements:
- Follow the defined project structure, roles, playbooks, inventory, and README
- Apply Ansible best practices during implementation
- Keep the solution safe, idempotent, and maintainable
- Include dry-run support, safety checks, and OS-specific handling where needed
- Add testing and validation guidance in the delivered files
- Ensure the final output is complete and ready to run
```
